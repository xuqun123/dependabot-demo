# Contributing to dependabot-demo

## Purpose

This repository is a demo for trying out [GitHub Dependabot](https://docs.github.com/en/code-security/dependabot). It contains intentionally outdated dependencies in JavaScript (npm) and Ruby (Bundler) so you can see how Dependabot opens pull requests for security and version updates.

## How to Contribute

1. **Fork** this repository.
2. **Create a branch** for your changes (`git checkout -b my-change`).
3. **Make your changes** and commit them with a clear message.
4. **Push** your branch and open a **Pull Request** against `main`.

## Code Style

This is a demo repository, so keep things simple:

- Follow existing conventions in the JavaScript and Ruby directories.
- Don't add unnecessary complexity -- the goal is to showcase Dependabot, not build production software.
- Keep dependency files (`package.json`, `Gemfile`) easy to read.

## Pull Request Process

1. Describe what your PR changes and why.
2. Make sure any new or modified files are consistent with the rest of the repo.
3. A maintainer will review and merge when ready.

## Questions?

Open an issue if you have questions about the repo or how Dependabot works with it.
